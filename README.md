# [虛擬健身教練]

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
自己平常有健身的習慣，但有時侯會怕自己的動作不對，但實際請教練不論花錢又要跑很遠，因此我想做一個虛擬健身教練，在維持動作品質的同時，量化自己的訓練成果。

### 預期功能
<!-- 列出預計實作的功能 -->
1.即時動作追蹤與品質分析。

2.紀錄訓練成果並以圖像化呈現。

3.客製化個人運動目標。

### 使用技術
<!-- 使用的語言、框架、工具等 -->
1.python

2.mediapipe(關節辨識)

3.opencv(影像)

4.NumPy (向量運算)

5.pandas、Matplotlib(圖像呈現)

### 時程規劃
<!-- 各週預計完成的進度 -->
Week7-8:熟悉套件並實作較簡單的動作(如伏地挺身)。

Week9-11:實作臥推、深蹲等較複雜動作，並確保有一定的精準度。

Week12-13:儲存訓練紀錄並以圖像化呈現。

Week 14-15：專案測試、評估完成度與Demo。

### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
1.OOP:因為會包含多個動作，所以可以建立一個Exercise的抽象父類別，並用各動作繼承。

2.向量運算:透過角度分析動作品質。
